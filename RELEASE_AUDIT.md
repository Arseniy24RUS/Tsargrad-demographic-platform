# Релизный аудит платформы «Россия 2050»

Проверка выполнена перед упаковкой проекта для GitHub Pages.

## Исправления расчётной логики

1. Страница «СКР»: разведены два разных показателя — отставание инерционного прогноза от цели и требуемый прирост после запуска мер. Дельты на графике и в карточках пересчитываются при смещении ползунка месяца ввода мер.
2. Страница «СКР»: федеральная целевая траектория больше не навязывается каждому субъекту как индивидуальная региональная норма; выбранная территория отображается как сравнительный контекст к России.
3. Страница «СКР»: подключён авторский годовой прогноз СКР из репозитория `Population-Forecast-Dashboard-of-Russia-2100`; при недоступности внешнего файла используется локальная экстраполяция.
4. Страница «Расселение»: базовая численность населения подтягивается из авторского когортно-компонентного прогноза с миграцией / без миграции при наличии доступа к внешнему репозиторию; локальная траектория остаётся резервной.
5. Страница «Ипотека»: кредит корректно разделён на льготную и рыночную части сверх лимита; платёж семьи считается как сумма двух платежей.
6. Страница «Ипотека»: аннуитетный метод сделан основным, режим простой переплаты сохранён как методический вариант.
7. Страница «Выплаты»: итоговый управленческий вывод вынесен отдельной строкой; технический параметр сохранения когорт спрятан в дополнительные настройки.
8. Страница «Усадьба»: расчётная площадь теперь равна сумме видимых модулей; при нехватке площади автоматически добавляется модуль общих пространств.

## Исправления интерфейса

1. Видимый интерфейс проверен на англоязычные слова: HTML-страницы не содержат видимых латинских слов.
2. Стандартная панель Plotly отключена, чтобы не появлялись англоязычные подсказки.
3. Служебные формулировки «прототип», «baseline», «debug», «TODO» удалены из видимого интерфейса.
4. Технические методические оговорки перенесены в раскрываемые блоки или README.
5. Навигация унифицирована на пяти страницах: СКР, Расселение, Усадьба, Ипотека, Выплаты.
6. Футер приведён к нейтральной формулировке «расчётная платформа для экспертного обсуждения».

## Визуальная полировка

1. Сохранён презентационный стиль: тёмный бирюзовый, золото, кремовые карточки, плотная типографика, геометрическая подача.
2. Страница «Усадьба» переведена с плоской кубической Canvas-схемы на трёхмерную сцену Three.js: дом с двускатными крышами, окнами, дверями, крыльцом, дорожками, участком, ограждением, деревьями и садом.
3. Визуализация усадьбы обновляется при изменении состава семьи и нормативов.
4. График потенциала матерей оставлен в исправленном виде: фактический столбик + штрихованная надстройка гипотетического максимума.

## Техническая проверка

1. Все JavaScript-файлы прошли синтаксическую проверку `node --check`.
2. Все JSON-файлы в папке `data/` успешно разбираются стандартным парсером JSON.
3. Проверено соответствие `id` в HTML и ссылок на элементы из JS: отсутствующих элементов не обнаружено.
4. Проверены видимые тексты HTML: латинских слов в интерфейсе не обнаружено.

## Оставленные проектные ограничения

1. Региональный слой РПН-2022 по субъектам РФ не включён, так как в открытых микроданных 2022 года отсутствует безопасный субъектовый ключ. Для регионального контекста используются РПН-2012/2017.
2. Сценарные расчёты не являются официальным прогнозом; это управленческая модель для экспертного обсуждения.
3. Трёхмерная визуализация использует Three.js из внешнего источника; при отсутствии интернета остаётся резервное сообщение загрузки, а расчётные графики продолжают работать.
