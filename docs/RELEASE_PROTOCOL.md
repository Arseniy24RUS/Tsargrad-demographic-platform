# Протокол релизной работы Codex

## Обязательный цикл

1. Прочитать `AGENTS.md`.
2. Прочитать все файлы `.codex/tasks/*.md`.
3. Составить план в `.codex/reviews/current_plan.md`.
4. Выполнять задачи по фазам.
5. После каждой фазы запускать проверки.
6. Исправлять ошибки до чистого результата.
7. Писать краткий отчёт саморевью.
8. Перед загрузкой на GitHub выполнять реальный Playwright QA на локальном сервере: desktop и mobile viewport, видимая отрисовка, console health и целевые интеракции.
9. Только после чистого релиза собирать архив.

## Фазы

1. Аудит текущего проекта.
2. Удаление внешних runtime-зависимостей.
3. Локализация авторского прогноза.
4. Разделение территориальных режимов.
5. Разгрузка первой страницы.
6. Доработка страниц Расселение, Усадьба, Ипотека, Выплаты.
7. Русификация UI.
8. Документация и лицензии.
9. Автотесты.
10. Сборка релиза.

## Правило остановки

Не останавливайся, пока не выполнены все пункты `docs/ACCEPTANCE_CRITERIA.md` или пока невозможно выполнить пункт из-за отсутствия исходных данных. В последнем случае ограничение должно быть явно записано в `RELEASE_CHECKLIST.md`.
